# project.yml Template

Standard template for xcodegen. Generated by `scaffold-project.sh`.

```yaml
name: AppName
options:
  bundleIdPrefix: com.company
  deploymentTarget:
    iOS: "26.0"
  xcodeVersion: "16.3"
  defaultActorIsolation: MainActor
settings:
  base:
    SWIFT_VERSION: "6"
    PRODUCT_BUNDLE_IDENTIFIER: com.company.appname
    INFOPLIST_FILE: AppName/Resources/Info.plist
    ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon
    SWIFT_STRICT_CONCURRENCY: complete
    SWIFT_APPROACHABLE_CONCURRENCY: YES
    CODE_SIGNING_ALLOWED: "NO"
targets:
  AppName:
    type: application
    platform: iOS
    sources:
      - path: AppName
    resources:
      - path: AppName/Resources
    settings:
      base:
        PRODUCT_BUNDLE_IDENTIFIER: com.company.appname
```

## Adding SPM Packages

In project.yml, packages are added under the `packages` key and referenced in target dependencies:

```yaml
packages:
  lottie-ios:
    url: https://github.com/airbnb/lottie-ios
    from: "4.4.0"
targets:
  AppName:
    dependencies:
      - package: lottie-ios
        product: Lottie
```

Use `mcp__xcodegen__add_package` to add packages without manually editing YAML.

## Build Settings Reference

| Setting | Value | Purpose |
|---|---|---|
| SWIFT_VERSION | 6 | Swift 6 language mode |
| SWIFT_STRICT_CONCURRENCY | complete | All data races are errors |
| SWIFT_APPROACHABLE_CONCURRENCY | YES | Better error messages |
| CODE_SIGNING_ALLOWED | NO | Simulator builds without signing |
| defaultActorIsolation | MainActor | Default to main actor |
